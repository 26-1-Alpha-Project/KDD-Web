# 6~8주차 활동 보고서 (프론트엔드)

## 1. 채팅 UI 컴포넌트 개발 (6주차 전반)

채팅 핵심 UI 개발에 착수했다. 메시지 버블과 입력창 컴포넌트를 먼저 구현하고, 이어서 답변 카드 컴포넌트에 출처 문서 링크와 신뢰도 배지를 통합했다. AI 답변이 어떤 문서의 몇 페이지를 참조했는지 사용자가 바로 확인할 수 있도록 SourceCard를 설계했으며, 로딩 스피너, 에러 알림, 대화 없음 등 빈 상태 UI도 함께 개발하여 채팅 화면의 모든 상태를 커버했다. SSE 스트리밍 수신 로직을 구현해 AI 답변이 실시간으로 한 글자씩 렌더링되는 체감을 확보했고, Mock 데이터 기반으로 전체 채팅 플로우를 완성하여 백엔드 없이도 시연 가능한 상태를 달성했다.

> **사진 추천**: 채팅 화면에서 AI 답변이 출처 카드(문서명+페이지)와 함께 표시된 모습, 신뢰도 배지("높은 신뢰도")가 보이는 스크린샷

## 2. FAQ 바로가기 및 부가 기능 UI (6주차 후반)

FAQ 페이지에서 자주 묻는 질문을 카테고리별로 탐색하고, 답변을 펼쳐볼 수 있는 아코디언 UI를 구현했다. 각 FAQ 항목에는 토픽 뱃지, NEW 표시, 작성 시간이 함께 노출되며, 도움이 됨/안 됨 피드백 버튼을 통해 사용자 반응을 수집하는 구조를 갖췄다. 파일 첨부 컴포넌트는 PDF 업로드를 대비하여 드래그앤드롭 영역과 파일 미리보기를 포함했고, 에이전트의 추가정보 요청 시 동적으로 입력 폼이 생성되는 UI도 프로토타입 수준으로 구현했다. 주간 스프린트 리뷰에서 전체 플로우를 시연하며 팀원 피드백을 반영했다.

> **사진 추천**: FAQ 페이지에서 카테고리 필터 탭과 아코디언이 펼쳐진 답변 화면, 피드백 버튼이 보이는 모습

## 3. AI 서버 연동 및 통합 테스트 (7주차)

배포된 AI 서버에 SSE 스트리밍 요청을 보내고 응답 파싱이 정상 동작하는지 검증했다. meta, text, done, fallback, error 다섯 가지 이벤트 타입을 각각 테스트하여 프론트에서 올바르게 분기 처리됨을 확인했다. 이어서 로컬 백엔드를 Docker로 구동한 뒤 프론트엔드-백엔드 간 E2E 통신 테스트를 수행하며, 인증 토큰 전달, 세션 생성, 메시지 전송 등 주요 흐름에서 발생한 CORS 및 타입 불일치 이슈를 수정했다. 통합 테스트 과정에서 발견된 UI 디테일(스크롤 위치, 메시지 순서, 타임스탬프 포맷)도 함께 보완하여 실제 서비스 수준의 안정성을 확보했다.

> **사진 추천**: SSE 스트리밍으로 답변이 실시간 타이핑되는 순간 캡처 (텍스트가 절반만 나온 상태), 개발자 도구 Network 탭에서 EventStream이 보이는 화면

## 4. 모바일 반응형 및 버그 트래킹 (7주차 후반)

중간 보고서 준비에 앞서 모바일 반응형 대응 상태를 전수 점검했다. 375px(iPhone SE)부터 1440px(데스크톱)까지 주요 뷰포트에서 사이드바 오버레이, 채팅 입력창 고정, FAQ 카테고리 탭 줄바꿈, 자료 카테고리 트리 스택킹 등을 검증하고 깨지는 레이아웃을 수정했다. 발견된 버그는 GitHub Issues에 재현 단계와 스크린샷을 첨부하여 등록했으며, 심각도별(Critical/High/Medium/Low) 라벨링으로 우선순위를 관리했다. 주간 회고에서 반응형 관련 반복 이슈를 분석하고, Tailwind의 h-dvh 사용이나 모바일 주소창 고려 등을 코딩 규칙에 추가했다.

> **사진 추천**: 모바일(375px)에서 사이드바 오버레이가 열린 채팅 화면, 데스크톱과 모바일을 나란히 비교한 반응형 스크린샷

## 5. 자료 페이지 구현 및 출처 연결 (8주차)

자료 페이지의 카테고리 트리 탐색 기능을 구현했다. CategoryTree 컴포넌트에서 상위-하위 카테고리를 트리 구조로 펼치고 접을 수 있도록 하고, 선택된 카테고리에 해당하는 문서 목록이 useDocuments 훅을 통해 실시간으로 필터링되도록 연결했다. DocumentCard에는 문서 제목, 카테고리, 업데이트 일자, 페이지 수 등 메타 정보를 표시하고, 클릭 시 DocumentDetail 페이지(/resources/[fileId])로 이동하여 PDFViewer 컴포넌트와 원문 다운로드 기능을 제공하는 구조를 완성했다. 채팅 답변의 출처 카드(SourceCard)를 클릭하면 해당 문서의 상세 페이지로 바로 이동하도록 라우팅을 연결하여, 채팅-자료 간 원활한 탐색 흐름을 확보했다.

> **사진 추천**: 자료 페이지에서 카테고리 트리가 펼쳐진 모습과 문서 목록, 출처 카드 클릭 시 문서 상세 페이지로 이동하는 흐름

## 6. 대화 세션 메모리 연동 및 알파 배포 (8주차)

대화 세션 메모리(Context) 연동을 위한 ChatContext 구조를 설계하고, 세션별 대화 이력이 유지되는 기반을 마련했다. 사용자가 이전 대화를 이어갈 수 있도록 SidebarContext와 ChatHistory 컴포넌트를 연결하여 세션 목록에서 과거 대화를 선택하면 해당 메시지가 복원되는 흐름을 구현했다. 동시에 Vercel을 통해 알파 테스트용 빌드를 배포하고, Lighthouse로 성능 프로파일링을 수행하여 LCP, CLS, INP 등 Core Web Vitals 지표를 측정했다. 1차 사용성 테스트 계획을 수립하고, 백엔드 API 수정 요청서를 작성하여 FAQ 응답 필드 추가, FAQ 피드백 API 신규 요청, 통계 응답 구조 확정 등 프론트-백엔드 간 인터페이스 불일치 사항을 정리하여 전달했다.

> **사진 추천**: Vercel 배포 화면 또는 배포된 URL 접속 모습, Lighthouse 성능 점수 리포트, 백엔드 API 수정 요청서 문서

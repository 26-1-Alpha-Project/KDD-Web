# 내 정보 수정

백 진행상황: 완료
설명: 현재 로그인한 사용자의 프로필 정보를 수정한다.
마이페이지에서 정보 수정 시 사용한다.
API 그룹: Users
URL: /users/me
METHOD: PATCH
인증 필요: Yes
기능명세서: 사용자 정보 수정 (https://www.notion.so/32cf1fecd5bc80b4bb51e50be54292ac?pvs=21)
담당자: 장민주
프론트 연결 상태: 시작 전
API 명세서 업데이트: 2026년 4월 10일

> 인증 필요: `Authorization: Bearer <accessToken>`
> 

### 📥 요청(Request)

**Request Body (JSON)** — 변경할 필드만 포함

```json
{
  "name": string,

  // 학생 전용
  "studentId": string,
  "department": "software" | "ai",
  "grade": number,
  "admissionYear": number,
  "academicStatus": "enrolled" | "on_leave" | "returning",
  "additionalInfo": string,

  // 교직원 전용
  "staffDepartment": "student_support" | "academic_affairs" | "admissions" | "industry_cooperation" | "international_office" | "general_affairs" | "other",
  "jobDescription": string
}

```

> 변경할 필드만 포함하여 전송 (부분 수정 가능)
> 

### 📤 응답(Response)

**성공 (200 OK)**

> GET /users/me와 동일한 형태
> 

**실패 (400)** — INVALID_INPUT

**실패 (400)** — PROFILE_NOT_COMPLETED

```json
{
"error": "PROFILE_NOT_COMPLETED",
"message": "프로필 입력이 완료되지 않은 사용자입니다."
}
```

**실패 (401)** — UNAUTHORIZED

**실패 (500)** — INTERNAL_SERVER_ERROR

### 📜 상세 설명

- **마이페이지에서 정보 수정** 시 사용 (profileCompleted가 true인 사용자)
- PATCH 방식이므로 **변경할 필드만** 보내면 된다
- `userType`, `email`은 변경 불가
- 응답은 수정 후 전체 사용자 정보를 반환한다
- 필드를 보내지 않으면 변경하지 않음
- 빈 문자열(`""`)을 보내면 해당 필드를 초기화(null)

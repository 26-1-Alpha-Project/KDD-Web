# 토큰 재발급

백 진행상황: 완료
설명: Refresh Token을 이용하여 새로운 Access Token을 발급한다.
API 그룹: Auth
URL: /auth/refresh
METHOD: POST
인증 필요: Yes
기능명세서: 토큰 재발급 (https://www.notion.so/32df1fecd5bc8024ac15f6ccbb63f031?pvs=21)
담당자: 장민주
프론트 연결 상태: 시작 전
API 명세서 업데이트: 2026년 4월 3일

### 📥 요청(Request)

> 별도 Request Body 없음
> 
> 
> Refresh Token은 **HttpOnly Cookie**에 저장되어 있으므로, 브라우저가 요청 시 자동으로 포함한다.
> 

### 📤 응답(Response)

**성공 (200 OK)**

```json
{
	"accessToken": "string"
}
```

> Refresh Token Rotation 적용: 새 Refresh Token이 Set-Cookie로 재설정되고, 기존 토큰은 **PostgreSQL auth_sessions에서 무효화(revoked_at 설정)** 된다.
> 

**실패 (401)** — INVALID_REFRESH_TOKEN

```json
{
	"error": "INVALID_REFRESH_TOKEN",
	"message": "유효하지 않거나 만료된 Refresh Token입니다."
}
```

**실패 (500)** — INTERNAL_SERVER_ERROR

### 📜 상세 설명

| 키 | 설명 |
| --- | --- |
| accessToken | 새로 발급된 JWT Access Token (만료: 30분, payload의 exp로 만료 시간 확인) |
- 클라이언트는 Access Token **만료 5분 전**에 백그라운드에서 이 엔드포인트를 호출한다.
- 서버는 Cookie의 Refresh Token을 SHA-256 해싱 후 PostgreSQL auth_sessions 테이블의 refresh_token_hash와 대조하여 검증한다.
- 재발급 실패(401) 시 로그인 페이지로 리다이렉트한다.(프론트 처리)
- 동시 요청 시 중복 refresh 방지를 위해 프론트에서 큐 처리가 필요하다. 다만 백엔드에서도 구현.

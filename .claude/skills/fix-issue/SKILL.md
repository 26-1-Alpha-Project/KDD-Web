# 이슈 해결

GitHub 이슈를 확인하고 해결 작업을 수행한다.

## 인자

- 이슈 번호 (필수): `/fix-issue 15`

## 단계

1. `gh issue view $ARGUMENTS`로 이슈 내용 확인
2. 이슈 유형에 맞는 브랜치 생성:
   - 기능: `feature/ET-번호-설명`
   - 버그: `fix/ET-번호-설명`
   - 리팩토링: `refactor/ET-번호-설명`
3. 관련 코드 탐색하여 영향 범위 파악
4. 구현 계획을 사용자에게 제시
5. 사용자 승인 후 작업 시작
6. 코드 수정/생성
7. `npm run lint` 통과 확인
8. 사용자에게 결과 보고

커밋은 `/commit`으로 별도 수행한다.

## 주의사항

- 작업 전에 반드시 구현 계획을 사용자에게 보여주고 승인받기
- 기존 컴포넌트/훅/유틸리티를 최대한 재사용
- 새 파일 생성 시 프로젝트 컨벤션 준수

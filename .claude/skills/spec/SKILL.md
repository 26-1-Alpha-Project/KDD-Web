# 스펙 문서 작성

기능 구현 전에 요구사항과 설계를 먼저 정리한다.
코드를 바로 짜지 않고, 사용자와 합의된 스펙 기반으로 구현하기 위함.

## 단계

1. 사용자에게 기능 요구사항 확인 (무엇을, 왜, 어디에)
2. 관련 코드베이스 탐색 (기존 패턴, 재사용 가능한 컴포넌트/훅)
3. 아래 형식으로 스펙 문서 작성
4. 사용자 확인 후 구현 시작

## 스펙 문서 형식

```markdown
# [기능명] 스펙

## 목적
왜 이 기능이 필요한지 한 줄

## 요구사항
- [ ] 구체적인 동작 1
- [ ] 구체적인 동작 2

## 영향 범위
- 수정할 파일:
- 새로 생성할 파일:

## 재사용할 기존 코드
- 컴포넌트:
- 훅:
- 유틸:
- 타입:

## UI 동작
- 사용자 인터랙션 흐름 설명

## API (해당 시)
- 요청/응답 타입 (types/api.ts에서 참조)
```

## 주의사항

- 스펙 문서는 코드가 아니라 **합의 도구**이다
- 사용자가 "OK" 할 때까지 구현에 착수하지 않는다
- 스펙에 명시되지 않은 기능을 임의로 추가하지 않는다

GitHub 이슈를 분석하고 코드베이스를 탐색하여 구현 계획을 수립한다.

## 워크플로우

1. `gh issue view <number>`로 이슈 내용 확인
2. 이슈 유형 분류: 기능 요청 / 버그 수정 / 리팩토링 / 문서
3. 관련 코드 탐색:
   - 영향받는 파일 식별
   - 관련 컴포넌트/훅/유틸리티 분석
   - 기존 패턴 확인
4. 구현 계획 작성
5. 적절한 브랜치명 제안

## 출력 형식

```
## 이슈 분석: #번호 — 제목

### 유형
기능 요청 / 버그 수정 / 리팩토링

### 영향 범위
- 수정/생성할 파일 목록

### 구현 계획
1. 단계별 설명

### 브랜치명
`feature/ET-xxx-description` 또는 `fix/ET-xxx-description`
```

한국어로 작성한다.
